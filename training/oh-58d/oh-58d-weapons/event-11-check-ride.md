## Check Ride Overview

The Weapons Employment Check Ride is the terminal evaluation of the OH-58D Kiowa Warrior WEC. It is a **pass/fail** event conducted as a complete mission from cold-and-dark startup through shutdown. The student is evaluated on all weapons and tactics knowledge gained across Events 1–10.

### Check Ride Philosophy

The check ride tests whether the student can conduct a **full weapons employment mission independently and safely**. The check pilot is an observer, not an instructor. The check pilot will:

- Observe and evaluate
- Ask oral questions during ground phases
- Inject simulated emergencies at designated points
- Not provide coaching, reminders, or hints during flight

### Grading System

| Grade | Definition |
|-------|------------|
| **PASS** | All mandatory standards met; student demonstrates safe, effective weapons employment |
| **REPEAT** | One or more mandatory standards not met; student must re-fly the check ride after additional training |
| **DOWN** | Unsafe act observed; immediate termination; additional training and review required before re-examination |

### Automatic REPEAT Standards

The following items, if failed, automatically result in a REPEAT regardless of overall performance:

| Item | Standard |
|------|---------|
| Laser released before impact (any weapon) | Automatic REPEAT |
| PRF code mismatch before guided weapon employment | Automatic REPEAT |
| Master Arm not set SAFE after each engagement | Automatic REPEAT (> 2 occurrences) |
| Hull-down time exceeded 90 seconds (consistent pattern) | Automatic REPEAT |
| Fratricide or self-damage from weapons employment | Automatic REPEAT |

### Automatic DOWN Standards

| Item | Standard |
|------|---------|
| Controlled flight toward terrain without immediate recovery | Automatic DOWN |
| Controlled flight into terrain (CFIT) | Automatic DOWN |
| Any unsafe aircraft handling that places crew or simulated personnel at risk | Automatic DOWN |

## Prerequisites for Check Ride

The student may not begin the check ride until:

- [ ] Events 1–10 are all marked PASS
- [ ] All Incomplete Items (IIs) from previous events are resolved
- [ ] The student has not accumulated more than 3 unresolved IIs at any point
- [ ] The student can brief the check pilot on the mission scenario without reference to course materials

The check pilot will conduct a **pre-check-ride oral** covering:
- Weapon system limitations (one random weapon system)
- PRF code deconfliction procedures
- Defensive priority sequence
- CCA vs. CAS distinction
- Loadout rationale for the assigned scenario

If the student cannot satisfactorily answer the oral questions, the check ride may be delayed until knowledge deficiencies are corrected.

## Mission Scenario Structure

The check pilot assigns a tactical scenario prior to the check ride. The student does not know the specific scenario until the ground planning phase. The scenario includes:

| Element | Details Provided |
|---------|-----------------|
| **Mission** | Type (scout/attack, deliberate attack, CCA support) |
| **Threat Environment** | Enemy order of battle — AD systems, armor, air threats |
| **Target Set** | Expected target types within the EA |
| **Friendly Situation** | Friendly unit positions, FEBA, FARP location |
| **Weather** | Scenario conditions (temperature, ceiling, wind) |
| **Time of Day** | Day or night scenario |

The student must plan the mission, select the loadout, configure the DCS Mission Editor (or use a pre-built scenario), and brief the plan before starting engines.

## Phase 1 — Mission Planning Brief

**Duration:** 20–30 minutes

The student presents a complete mission plan to the check pilot, covering all elements from Event 10:

### Mandatory Brief Items

| Item | Standard |
|------|---------|
| **Threat ring map** | All threat systems plotted at correct ranges; ingress/egress routes avoid threat rings |
| **Loadout selection** | Stated with rationale (target set, threat, mission type, performance) |
| **Power check** | Computed for scenario conditions; HOGE capability confirmed or limitation stated |
| **Engagement plan** | EA boundaries; minimum BP (Primary + 2 Alternate); TRPs; trigger lines; engagement priorities |
| **Ammunition management** | Bingo states defined for each weapon; abort criteria stated |
| **Communication plan** | Frequencies and PRF code matrix briefed |
| **Weather impact** | Sensor and weapon effects for scenario conditions briefed |
| **Brevity codes** | Student demonstrates knowledge of standard brevity; kneeboard card referenced |

**Oral questions** are asked during or after the brief. The check pilot evaluates knowledge depth, not just memorized checklists.

## Phase 2 — Startup & Systems Verification

**Standard:** Cold-and-dark; student completes startup without instructor prompting.

| Evaluation Item | Standard |
|----------------|---------|
| Startup procedure | Completed per BQT standards; no memory aids required |
| MFD power-on and navigation | All relevant pages configured |
| MMS power-on | MMS powered; boresight completed before taxi; LRF range confirmed |
| Weapons page verification | Loadout confirmed matches planned loadout; round counts verified |
| PRF codes verified | MMS PRF matches weapon PRF on each weapons sub-page |
| HOTAS binding verification | Student verbally confirms chaff, flare, laser, MMS slew, sensor toggle bound and tested |
| Master Arm | Confirmed SAFE at startup; not armed until entering attack profile |

---

## Phase 3 — Ingress & Countermeasures

**Standard:** Student navigates from FARP to assigned BP area using a terrain-masked route, without instructor guidance.

| Evaluation Item | Standard |
|----------------|---------|
| Route selection | Ingress follows terrain-masked corridor; avoids threat ring boundaries plotted in planning |
| Altitude discipline | NOE / contour flight in threat environment (25–200 ft AGL as appropriate) |
| RWR monitoring | Student acknowledges every RWR contact; states threat type and bearing when queried |
| Countermeasures response | Correct CM type (chaff or flare) dispensed for simulated threat type called by check pilot |
| Terrain masking discipline | Aircraft masked within 5 seconds of any simulated "locked" call from check pilot |
| No-fly zone compliance | Aircraft does not enter threat ring boundaries without stated plan and justification |

**Simulated threat injection:** The check pilot calls a simulated threat (e.g., "SA-8 lock" or "MANPADS launch") at one point during ingress. The student must respond with the correct defensive sequence without prompting.

---

## Phase 4 — Battle Position Operations

**Standard:** Student selects and occupies a valid BP for the assigned EA; executes hull-down engagement cycle within time standard.

| Evaluation Item | Standard |
|----------------|---------|
| BP selection | BP provides fields of fire to EA; terrain masking available; covered approach route used |
| Hull-down execution | MMS clears mask; fuselage remains below cover; ≤ 60 seconds per unmask cycle |
| MMS operation | WFOV scan → NFOV identification; TV/TIS selection appropriate for conditions |
| Target identification | PID conducted before engagement declared; target type stated to check pilot |
| Displacement | Aircraft moves to alternate BP after each 1–2 engagements from primary; without prompting |
| Approach/egress | No skylining; covered routes used; approach and egress routes different |

---

## Phase 5 — Weapons Employment

**Standard:** Student must employ **at minimum two different weapon systems** during the check ride. The specific weapons employed depend on the assigned scenario loadout.

### For Each Weapon System Employed

| Evaluation Item | Standard |
|----------------|---------|
| Pre-attack checks | Weapons page verified; warhead/variant correct; PRF code confirmed (guided weapons) |
| Master Arm sequence | ARM when entering attack; SAFE immediately after last engagement |
| Weapon-specific procedure | Correct employment sequence per Events 3–6 standards |
| Laser discipline (guided weapons) | Laser active before fire; maintained through impact; no early release |
| CCIP discipline (unguided) | Pipper stabilized before fire; burst discipline for gun; ranging shot for rockets |
| BDA | Conducted after every engagement; results reported within 15 seconds |
| Re-engagement / target selection | Correct engagement priority (AD → armor → C2 → soft) when multiple targets present |
| Ammunition management | Bingo state reported when reached; student initiates without prompting |

### Minimum Weapons Employment Standards

| Weapon | Minimum Hits Required | Notes |
|--------|----------------------|-------|
| **AGM-114 Hellfire** (if loaded) | 2 of 4 shots must be direct hits | LOBL standard: lock before fire; laser through impact |
| **Hydra 70 Rockets** (if loaded) | 1 direct hit per 3 rockets at ≤ 2,000 m | Ranging shot expected |
| **APKWS** (if loaded) | 3 of 4 rounds must be direct hits | CCIP + laser; PRF verified |
| **M3P .50 cal** (if loaded) | Tracer impacts within 10 m; 2 of 3 passes with observed hits | Burst discipline: ≤ 3 seconds per burst |

## Phase 6 — CCA Tactics & Target Handoff

**Standard:** Student demonstrates application of CCA principles in a multi-target or simulated team scenario.

| Evaluation Item | Standard |
|----------------|---------|
| Engagement priority | Correct priority applied when multiple target types present (AD first) |
| Hull-down time standard | ≤ 60 seconds per unmask cycle maintained throughout Phase 5 and 6 |
| Target handoff | Student passes one target to check pilot (simulating wingman) using TRP-based format: "From TRP [X], [direction], [distance], [description]" |
| Talk-on (if called) | Student correctly talks check pilot (simulating ground element) onto a target using standard sequence |
| Sectors / deconfliction | If multi-ship scenario: student correctly assigns sectors and coordinates without confusion |
| Pop-up attack (if called) | If assigned by check pilot: apex time < 15 seconds; IP and egress briefed before execution |

---

## Phase 7 — Emergency Procedures

**Standard:** Check pilot calls one simulated emergency during Phase 5 or 6. Student must respond correctly without coaching.

**Possible injected emergencies:**

| Emergency | Expected Response |
|-----------|-----------------|
| **"Weapons misfire"** | Correct misfire/hangfire procedure per Event 2 (safe weapon; select next weapon; jettison if necessary) |
| **"Master Arm failed — won't ARM"** | Troubleshoot per MFD BIT; report status to check pilot; identify alternate options |
| **"JTAC calls Abort"** | Immediate trigger release; Master Arm → SAFE; break off; report acknowledgment |
| **"Simulated SA-8 launch"** | Correct defensive sequence: chaff + break turn + terrain mask; CM HOTAS used (not panel) |
| **"Hung ordnance"** | Return to FARP; avoid jettison zones; notify FARP of hung ordnance; land safely |
| **"Bingo fuel — 20 minutes remaining"** | Report to check pilot; begin egress; confirm FARP routing |

---

## Phase 8 — Egress & Shutdown

**Standard:** Student conducts terrain-masked egress and safe shutdown.

| Evaluation Item | Standard |
|----------------|---------|
| Egress route | Different from ingress; terrain-masked; avoids threat rings |
| FARP approach | Correct approach to assigned FARP |
| Weapons safe | All weapons confirmed SAFE before landing; ordnance status reported to FARP |
| Shutdown | Completed per BQT shutdown procedure |
| Debrief | Student leads self-debrief covering: hits/misses, errors observed, what would be done differently |

## Common Reasons for REPEAT or DOWN

### REPEAT

| Reason | Prevention |
|--------|-----------|
| Laser released before impact | Drill the standard in every training event: laser releases AFTER "Splash" call |
| Hull-down time > 90 seconds on multiple occasions | Practice the cycle until 60 seconds is automatic; instructor stops student during training the moment 60 seconds is reached |
| Poor engagement priority (shooting soft targets before AD/armor) | Brief and rehearse the priority table; make it a habit to call "Priority One — Air Defense" before scanning for targets |
| PRF code not verified before employment | Write PRF on kneeboard; verbal confirmation step in every pre-attack check |
| No alternate BP displacement | Displacement must be automatic; train with instructor calling "Move" after every second engagement until the habit is formed |
| CM not HOTAS-mapped | Verified during preflight; any unmapped CM button is a training deficiency to correct before the check ride |

### DOWN

| Reason | Prevention |
|--------|-----------|
| CFIT during NOE / attack profile | Strict altitude discipline in training; instructor terminates any training flight where terrain margin drops below 15 ft AGL |
| Break-off altitude/range violated on diving attack | Hard briefed limits; training instructor calls "Pull off" if student reaches limit; no "one more second" |
| Unsafe aircraft handling under target fixation | Train systematic MFD cross-check during MMS operations; never fly on instruments only through MMS eyepiece |
