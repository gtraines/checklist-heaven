## Course Description

The OH-58D Kiowa Warrior Basic Qualification Training (BQT) course qualifies a pilot to safely operate the Bell OH-58D Kiowa Warrior in day VMC conditions from cold-and-dark startup through safe aircraft recovery. Combat employment (weapons delivery, Hellfire engagement, Hydra 70 rocket employment, .50 cal gunnery, RSTA mission tactics, scout/attack team procedures) is **not** covered in this course.

> **DCS Module Note:** The DCS OH-58D (Polychop Simulations) is a **full-fidelity, clickable cockpit** module with an Advanced Flight Model (AFM). Every switch, dial, and control is interactive. All procedures in this course involve physical cockpit interaction via mouse click. The module uses a realistic rotor dynamics model — hover requires active pilot control throughout; the aircraft will not hover hands-off.

Upon successful completion, the student will be qualified to:
- Perform an independent full cold-and-dark startup including engine start, AHRS/GPS alignment, and rotor engagement
- Perform a stationary hover and hover taxi with controlled translating tendency compensation
- Execute a normal takeoff including transition through ETL and climb departure
- Conduct basic airwork: coordinated turns, climbs, descents, slow flight, trim technique
- Navigate between waypoints using the AHRS/GPS, MFD moving map, and HSI
- Fly a standard VFR helicopter traffic pattern and execute multiple approach types including go-arounds
- Respond correctly (verbally and procedurally) to all boldface emergency procedures
- Perform a practice autorotation to a safe recovery
- Complete a controlled full shutdown from engine running to cold-and-dark

## Prerequisites

Before beginning this course, the student must:
- [ ] Have completed **basic rotary-wing familiarization** or demonstrate hover proficiency in any DCS helicopter (UH-1H, Mi-8, Ka-50, etc.) — **this is not a first-helicopter course**
- [ ] Understand basic DCS World controls: external view, F10 map, communications menu, pause/unpause
- [ ] Have a functional HOTAS configured with collective axis, cyclic axes (pitch/roll), and anti-torque pedals or twist-rudder
- [ ] Have read the **DCS OH-58D Polychop documentation** (in-game manual or PDF)
- [ ] Be able to use the mouse to click cockpit switches in DCS (hover, left-click, scroll wheel for rotary knobs)

> **Recommended:** Achieve a stable unassisted 3-foot hover for 30 seconds before attempting Event 2. If this is not achievable, practice hover control with the DCS free flight option before proceeding.

> **Note on Pedals:** Anti-torque control is critical in the OH-58D. A twist-rudder joystick is the minimum recommended setup. Dedicated rudder pedals are strongly preferred. Keyboard rudder inputs (`Z`/`X`) are not recommended for this aircraft — the discrete key steps make precise yaw control extremely difficult.

## Syllabus Events

Complete events **in sequence**. Each event must pass before progressing.

| Event | Title | Type | Description |
|---|---|---|---|
| [Event 1](event-1-academics.md) | Aircraft Systems & Academics | Ground | Systems, cockpit layout, limitations, boldface emergency procedures |
| [Event 2](event-2-startup-ground-ops.md) | Startup & Ground Operations | Flight | Cold-dark startup, AHRS/GPS alignment, rotor engagement, hover taxi, shutdown |
| [Event 3](event-3-hover.md) | Hover & Hover Taxi | Flight | Stationary hover, pedal turns, hover taxi, translating tendency, power check |
| [Event 4](event-4-fam-flight.md) | Familiarization Flight | Flight | Normal takeoff, basic airwork, trim technique, normal approach and landing |
| [Event 5](event-5-navigation.md) | Basic Navigation | Flight | AHRS/GPS waypoints, MFD moving map, HSI, cross-country nav profile |
| [Event 6](event-6-traffic-pattern.md) | Traffic Pattern & Landings | Flight | VFR pattern, multiple approach types, go-arounds, slope/confined area intro |
| [Event 7](event-7-emergency-procedures.md) | Emergency Procedures | Flight | Simulated emergencies, practice autorotation, boldface execution |
| [Event 8](event-8-check-ride.md) | Check Ride | Evaluation | Full-profile pass/fail evaluation |

**Estimated Completion:** 8–14 simulator sessions.

> **Note:** Events 2 through 8 each begin from cold-and-dark unless otherwise noted. Autostart is **not permitted** during graded events — the student must demonstrate the full manual startup. Autostart may be used for self-study only.

## Grading Standards

### Grades

| Grade | Definition |
|---|---|
| **PASS** | All standards met; student may progress to the next event |
| **REPEAT** | One or more standards not met; event must be re-flown before progressing |
| **DOWN** | Unsafe act observed; event grounded pending debrief and instructor review |

### Incomplete Item (II)

An Incomplete Item (II) is issued when the student fails one specific standard but otherwise passes the event. All IIs must be resolved **before the Check Ride (Event 8)**. A student accumulating more than three unresolved IIs is considered not ready for the check ride.

### Grading Tolerances

| Maneuver | Standard |
|---|---|
| **Stationary hover (< 5 kts wind)** | ±2 ft altitude, ±3 ft lateral/longitudinal drift over 30 sec |
| **Hover altitude (general)** | 3–5 ft skid height maintained |
| **Pedal turn** | ±5° of target heading, ±2 ft altitude, no drift |
| **Level flight** | ±100 ft altitude, ±5 KIAS |
| **Coordinated turn** | Ball centered throughout; ±100 ft altitude |
| **Autorotation entry Nr** | 90–107% Nr maintained at entry; no Nr exceedance |
| **Approach angle (normal)** | 8–12° glidepath; deceleration to hover at designated point |
| **Approach angle (steep)** | 14–20° glidepath maintained |
| **Hover to touchdown** | Smooth set-down; no ground resonance; both skids simultaneously |
| **Slope landing** | ≤ 10° side slope; upslope skid touches first |
| **Engine start (TOT)** | Must not exceed 843°C start limit; monitor throughout |
| **AHRS/GPS alignment** | Full alignment complete before hover taxi |
