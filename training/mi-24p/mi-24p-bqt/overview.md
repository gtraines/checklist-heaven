## Course Description

The Mi-24P Hind Basic Qualification Training (BQT) course qualifies a pilot to safely operate the Mil Mi-24P Hind-F in day VMC conditions from cold-and-dark startup through safe aircraft recovery. Weapons employment (ATGM, rockets, cannon, air-to-air missiles) is **not** covered in this course. Basic familiarization with the weapons panel layout, PKV sight, and SPO-15 RWR indicators is included for cockpit awareness only.

> **DCS Module Note:** The DCS Mi-24P (Eagle Dynamics — native module) is a **full-fidelity, dual-seat, clickable cockpit** module with an Advanced Flight Model (AFM). All switches in both the Pilot (rear) and WSO (front) cockpits are interactive. This course is structured for single-player using DCS's seat-switching feature (`1` = WSO front, `2` = Pilot rear). Multiplayer dual-seat operations with a separate WSO are fully compatible with this syllabus.

Upon successful completion, the student will be qualified to:
- Perform an independent full cold-and-dark startup in both cockpits including AI-9V APU start, engine start sequence, generator connection, avionics alignment, and rotor engagement
- Conduct a pre-takeoff check and perform running takeoff and hover takeoff profiles
- Perform a stable hover and hover taxi (with awareness of the Mi-24P's high power demand at low speed)
- Execute basic airwork: level flight, coordinated turns, climbs, descents, slow flight, force trim technique, and autopilot use
- Navigate using the DISS-15 Doppler, ARK-15 ADF, RSBN system, and HSI
- Respond correctly (verbally and procedurally) to all boldface emergency procedures, including single-engine failure, autorotation, and tail rotor failure
- Fly a standard VFR traffic pattern and execute running and hover landings, steep approach, and go-arounds
- Complete a controlled full shutdown from engine running to cold-and-dark in both cockpits

## Prerequisites

Before beginning this course, the student must:
- [ ] Have completed **basic rotary-wing familiarization** or demonstrate basic helicopter flight in any DCS helicopter — **this is not a first-helicopter course**
- [ ] Understand basic DCS World controls: external view, F10 map, comms menu, pause/unpause, seat switching (`1`/`2`)
- [ ] Have a functional HOTAS configured with collective axis, cyclic axes (pitch/roll), and anti-torque pedals or twist-rudder
- [ ] Have read the **DCS Mi-24P Eagle Dynamics module documentation** (in-game manual or PDF)
- [ ] Be able to use the mouse to click cockpit switches in DCS (left-click, scroll wheel for rotary knobs)

> **Recommended:** Complete at least 1 hour of hover practice in the UH-1H or Mi-8 before beginning Event 3. The Mi-24P's poor hover performance at high gross weight will frustrate a student with no prior hover experience.

> **Note on Controls:** Anti-torque pedals or a twist-rudder joystick are **required** for this aircraft. Keyboard rudder inputs (`Z`/`X`) cannot provide the fine control needed, especially at low speeds. The Mi-24P's right-side pusher tail rotor requires deliberate left-pedal input to counter torque during power increases.

> **Note on Gross Weight:** This course is flown at **training weight** (~10,000–10,500 kg: full fuel, no weapons). Do not load weapons for BQT events. Higher gross weight will reduce hover performance and may make hover landings impossible.

## Syllabus Events

Complete events **in sequence**. Each event must pass before progressing.

| Event | Title | Type | Description |
|---|---|---|---|
| [Event 1](event-1-academics.md) | Aircraft Systems & Academics | Ground | Systems, cockpit layout, limitations, boldface emergency procedures |
| [Event 2](event-2-startup-ground-ops.md) | Startup & Ground Operations | Flight | Cold-dark startup (both cockpits), APU start, engine start, avionics, hover taxi, shutdown |
| [Event 3](event-3-hover-takeoff.md) | Hover & Takeoff | Flight | Hover technique, power check, hover taxi, running takeoff, hover takeoff, climb profiles |
| [Event 4](event-4-fam-flight.md) | Familiarization Flight | Flight | Level flight, coordinated turns, climbs/descents, slow flight, force trim, autopilot use |
| [Event 5](event-5-navigation.md) | Basic Navigation | Flight | DISS-15, ARK-15 ADF, RSBN, HSI, radar altimeter, cross-country nav profile |
| [Event 6](event-6-emergency-procedures.md) | Emergency Procedures | Flight | Simulated emergencies, practice autorotation, boldface execution |
| [Event 7](event-7-traffic-pattern.md) | Traffic Pattern & Landings | Flight | VFR pattern, running landing, hover landing, steep approach, go-arounds, shutdown |
| [Event 8](event-8-check-ride.md) | Check Ride | Evaluation | Full-profile pass/fail evaluation — cold-dark to cold-dark |

**Estimated Completion:** 10–16 simulator sessions.

> **Note:** Events 2 through 8 each begin from cold-and-dark unless otherwise noted. Autostart (`LWin + Home`) is **not permitted** during graded events — the student must demonstrate the full manual startup procedure. Autostart may be used for self-study and practice only.

> **Dual-Seat Note:** If flying with a crewed WSO, the WSO is responsible for front cockpit items during startup, weapons safe verification, SPO-15, and UV-26. In single-player, the pilot switches seats (`1`/`2`) and completes all items.

## Grading Standards

### Grades

| Grade | Definition |
|---|---|
| **PASS** | All standards met; student may progress to the next event |
| **REPEAT** | One or more standards not met; event must be re-flown before progressing |
| **DOWN** | Unsafe act observed; event grounded pending debrief and instructor review |

### Incomplete Item (II)

An Incomplete Item (II) is issued when the student fails one specific standard but otherwise passes the event. All IIs must be resolved **before the Check Ride (Event 8)**. A student accumulating more than three unresolved IIs is not ready for the check ride.

### Grading Tolerances

| Maneuver | Standard |
|---|---|
| **Hover (OGE, < 5 kts wind)** | ±1 m altitude, ±3 m lateral/longitudinal drift over 30 sec |
| **Hover altitude (general)** | 3–5 m wheel height maintained |
| **Pedal turn in hover** | ±5° of target heading, ±1 m altitude, no drift |
| **Running takeoff** | Smooth acceleration; airborne before 200 m; no settling |
| **Level flight** | ±50 m altitude, ±10 km/h airspeed |
| **Coordinated turn** | Ball centered throughout; ±50 m altitude; bank ≤ 45° |
| **Climb / Descent** | Within ±2 m/s of target rate; airspeed ±10 km/h |
| **Autorotation Nr (entry)** | 89–105% Nr maintained; no Nr exceedance |
| **Running landing rollout** | Centerline ±5 m; no drift at touchdown; controlled rollout |
| **Hover landing** | Within ±2 m of intended point; controlled set-down |
| **Go-around** | Positive climb established within 5 sec; no secondary descent |
| **Engine start (EGT)** | Must not exceed 900°C starting limit; monitored throughout |
| **Crosswind operations** | Demonstrated in crosswind ≤ 12 m/s |
